# 系统基础与全链路：功能设计

> 实现标记：本文描述目标设计；当前 `0.1.0` 已实现项和后续边界以 [后端实现状态](../../implementation-status.md) 为准。

## 1. 目标

定义用户如何理解和使用 Nuntius，以及所有子模块共同遵循的业务边界。该模块不直接提供单一页面功能，而是保证设备、项目、会话和实时控制形成一条一致的产品路径。

## 2. 产品心智模型

用户始终按以下层级操作：

```text
我的账户
└─ 我的设备
   └─ 设备上的项目
      └─ 项目中的 Thread
         └─ Thread 中的 Turn、消息、执行与审批
```

用户不需要理解公网路由、App Server 进程或消息重放等内部概念；这些只在诊断页面中以健康状态呈现。

## 3. 角色

第一版只有一个业务角色：

- **Owner**：账户所有者，可以配对和撤销设备、管理项目、控制所有 Thread、处理审批和查看诊断。

第一版不设计团队、组织、访客和复杂 RBAC。数据结构保留 `user_id` 归属，避免未来迁移困难。

## 4. 核心功能地图

### 4.1 初次使用

1. 部署公网 Server。
2. 用户创建或初始化 Owner 账户。
3. 在工作电脑安装 CLI。
4. 使用一次性配对码注册设备。
5. CLI 检测并启动 Codex App Server。
6. 用户添加一个本地项目目录。
7. 手机登录远程页面，看到设备和项目。
8. 在项目中创建第一条 Thread 并发送消息。

### 4.2 日常使用

- 在设备之间切换。
- 在设备内选择项目。
- 新建或恢复 Thread。
- 发送消息并观察实时执行。
- 在等待审批时通过手机决定。
- 断网后等待自动恢复，或按页面提示重新同步。

### 4.3 本地独立使用

公网 Server 不可用时：

- 本地控制台仍可查看项目和 Thread。
- 仍可创建 Turn、查看事件和处理审批。
- 公网连接恢复后同步未确认实时事件，并继续将完整规范化会话历史回填到 Server；本地执行不依赖回填完成。

## 5. 全局业务规则

1. 一个 Project 必须且只能属于一台 Device。
2. 一个 Thread 必须且只能关联一台 Device 和一个 Project；无法映射 cwd 时归入该设备 system unassigned Project。
3. 远程操作必须指明目标 Device，禁止使用“当前任意在线设备”。
4. Device 离线时不接受会产生本地副作用的新命令。
5. 页面展示实时设备状态缓存时必须标记 `stale` 和最后更新时间；Server 已持久化历史另用 `complete/backfilling/partial` 表达完整性。
6. HTTP 返回“已接受”只表示命令已安全保存，不代表 Codex 已完成。
7. 操作超时不等于失败；系统必须查询命令状态。
8. 无法判断是否已执行的操作显示 `unknown`，不能自动重复。
9. 移除项目只删除 Nuntius 索引，不删除项目目录。
10. 撤销设备立即阻止新的远程控制，不删除该设备本地项目和会话。

## 6. 全局状态体验

### 6.1 设备状态

- 在线：可接受新命令。
- 同步中：已连上 Server，但暂不接受危险命令。
- 降级：Agent 在线，但 App Server 或本地存储异常。
- 离线：可查看 Server 已同步的完整历史，执行能力不可用，并标记最后同步时间。
- 已撤销：从正常设备列表移除，不可重新连接，需重新配对。

### 6.2 命令状态

- 已接受：Server 已持久化。
- 等待设备：尚未被设备确认。
- 已送达：设备已持久化。
- 执行中：已交给本地领域处理器或 App Server。
- 已完成、失败、拒绝：明确终态。
- 未知：无法安全判断结果，需要同步或用户处理。

### 6.3 页面新鲜度

- `live`：来自当前连接和最新快照。
- `syncing`：正在恢复事件或快照。
- `stale`：来自缓存。
- `complete/backfilling/partial`：Server 历史同步完整性，与设备是否在线分离。
- `unknown`：某个具体操作结果不确定。

## 7. 异常体验原则

- 错误说明“哪一段出了问题”：浏览器、Server、设备连接、Agent、App Server 或项目目录。
- 错误提供可执行动作：重试同步、打开诊断、重新登录、升级 CLI、在本机处理。
- 不显示虚假的“发送成功”；区分已接受、已送达和已执行。
- 不因局部事件丢失而清空整个对话；先请求快照。
- 不在网络恢复时自动执行已经过期的用户命令。

## 8. 第一版不做

- 团队共享设备。
- 跨设备同步项目文件。
- 自动迁移 Thread 到另一台设备。
- 多设备并行执行同一 Turn。
- 对完整历史建设复杂语义搜索、向量索引或跨设备自动合并。
- 复杂工作流编排和定时任务。
- 在手机上提供完整 IDE 或任意文件管理器。

## 9. 验收标准

1. 用户可从设备列表进入项目，再进入 Thread，路径始终明确。
2. 任意页面都能识别当前目标 Device 和 Project。
3. 设备离线时所有副作用按钮不可用并说明原因。
4. Server 断开后本地控制台仍可正常操作本机 Codex。
5. 远程页面不会把缓存状态误标为实时状态。
6. 命令的已接受、已送达、执行中和终态可区分。
7. 所有 `unknown` 状态都有重新同步或本地核对入口。
8. 未归类 Thread 仍可远程阅读历史，但不展示 cwd 且不能执行。
