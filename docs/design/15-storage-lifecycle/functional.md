# 数据存储与生命周期：功能设计

> 实现标记：本文描述目标设计；当前 `0.1.0` 已实现项和后续边界以 [后端实现状态](../../implementation-status.md) 为准。

## 1. 目标

明确哪些数据保存在设备、哪些汇总到 Server、保留多久、如何清理和恢复。系统采用“双层权威”：设备负责路径、文件与可执行 Codex 状态；Server 负责跨设备可查询的规范化完整历史。

## 2. 数据归属

### 设备是执行与本地文件权威

- Project 完整路径和本地配置。
- Codex 可恢复的原始 Thread、Turn 和 Item 状态。
- 文件变化和项目文件。
- App Server 映射。
- Agent inbox/outbox 和短期事件日志。

### Server 是身份、控制与全局历史权威

- Owner 账户和 Web Session。
- Device Registry、公钥和撤销状态。
- 配对码。
- 已接受远程 Command 状态。
- Device -> Project -> Thread -> Turn -> Item 的规范化索引与完整已同步历史。
- 用户消息、完整 Agent 消息、审批结果及受控的工具/命令/文件变更记录。
- 每台设备、项目和 Thread 的历史同步 checkpoint、完整度与最近同步时间。
- 安全审计元数据。

### 仅作短期状态或缓存的数据

- 活跃 Turn 状态。
- 待审批安全摘要。
- 原始 token/命令输出 delta 和 SSE replay cursor。
- 在线状态和实时连接状态。
- 目录选择器的 live query 结果；默认不长期缓存目录树。

## 3. 用户数据体验

- 设备离线时，页面仍可读取 Server 已同步的完整历史，并明确显示最后同步时间与 `complete/backfilling/partial`。
- 离线时不能新建 Turn、审批、Interrupt、刷新目录或创建 Project。
- 移除 Project 不删除文件。
- 归档 Thread 不删除会话。
- 撤销 Device 不删除设备本地数据。
- Server 数据丢失不应导致设备项目或 Codex 原始会话丢失；恢复或重新配对后可以从设备幂等回填。
- Server 侧删除历史、解除设备与删除本地 Codex 数据是三个不同动作，不得用一个含糊的“删除”入口合并。

## 4. 保留策略

第一版建议默认：

- Web Session：到过期或撤销后短期审计保留。
- Pairing Code：过期/使用后保留短期审计再清理。
- Command 元数据：保留 30 天，可配置。
- Agent Replayable Events：至少 24 小时并受磁盘上限约束。
- Server 规范化完整历史：默认长期保留，直到 Owner 明确删除或实例保留策略到期。
- Server 原始 delta：在完整 Item 已提交且重放窗口结束后清理，不作为长期历史格式。
- 诊断日志：滚动保留 7～14 天，可配置。
- Device/Project/Thread 索引：直到用户删除；解除/归档后保留 tombstone 窗口以支持去重和审计。

保留时间是默认工程参数，不是不可变业务承诺。

## 5. 删除和清理

- 删除动作必须区分索引删除、本地内容删除和审计清理。
- 第一版远程页面不提供永久删除本地 Codex Thread。
- 删除 Server 历史必须先说明影响范围，并以软删除 + 延迟物理清理实现；不连带删除设备文件。
- 清理任务必须先处理已确认、可重建数据。
- 磁盘不足时暂停新副作用，不能牺牲未确认命令。

## 6. 备份与恢复

Server：

- 定期 Server SQLite 备份。
- 必须验证恢复，不只验证备份文件存在。
- 恢复后设备重新连接，校正索引并从 checkpoint 幂等补齐完整历史。

Agent：

- SQLite 是本地运行状态，不要求上传云备份。
- 数据库损坏时先备份损坏文件，再尝试恢复/重建索引。
- Codex 完整历史不应因 Nuntius SQLite 损坏而删除。

## 7. 隐私

- Server 的完整历史天然包含敏感 prompt、回复以及部分执行记录，部署者必须把 Server SQLite、备份和导出都按敏感数据保护。
- 默认不上传可直接使用的绝对项目路径；显示名与不可逆 path hint 足够用于跨设备识别。
- 不上传项目源文件正文。命令输出和 diff 只保存可理解会话所需、经过大小限制与脱敏策略允许的内容；超限内容保存截断标记和本地引用状态。
- 日志和审计不得复制完整消息正文、Bearer Token、`directory_ref` 或命令输出。
- 用户可以查看 Server 当前保存的数据类别。
- HTTP/WS 模式下这些数据会明文经过网络，页面必须持续显示风险，不能用“已登录”暗示传输安全。

## 8. 验收标准

1. 数据归属与页面同步完整度、stale 标识一致。
2. 项目移除、设备撤销和 Thread 归档都不删除本地文件。
3. Server 备份恢复后设备能重新同步索引和缺失完整历史。
4. Agent 数据库损坏不删除 Codex 原始会话。
5. 磁盘压力清理不删除未确认 Durable 消息。
6. Server 不长期保存原始 Agent delta，但能从规范化 Item 读取完整消息。
7. 所有清理任务可观察并有失败重试。
8. 历史删除不删除项目文件或设备侧 Codex 原始状态。
9. 用户可在设备离线时读取全部已同步历史，并看见是否仍有回填缺口。
