# 用户身份与访问控制：功能设计

> 实现标记：本文描述目标设计；当前 `0.1.0` 已实现项和后续边界以 [后端实现状态](../../implementation-status.md) 为准。

## 1. 目标与范围

为远程控制页面和设备配对提供简单、安全、可撤销的身份体系。第一版只服务单个 Owner，不设计组织和团队权限。

本模块负责：

- Owner 登录、退出和会话管理。
- 初次账户初始化。
- 设备一次性配对授权。
- Web Session 与 Device Credential 分离。
- 撤销全部或指定登录会话。

设备名称、在线状态和能力归设备管理模块负责。

## 2. 初次初始化

### 2.1 未初始化状态

- Server 首次启动后处于 `uninitialized`。
- 只允许从明确配置的初始化路径创建第一个 Owner。
- 初始化完成后该入口永久关闭，除非管理员在服务器本机执行恢复命令。

### 2.2 初始化方式

第一版提供两种方式，部署时二选一：

- Server 启动时通过环境变量提供一次性初始化令牌。
- 在服务器终端运行管理命令创建 Owner。

禁止公开默认密码。

## 3. 登录

- 用户使用邮箱/用户名和密码登录。
- 登录成功后建立 HttpOnly Cookie 会话；HTTPS 档位同时使用 Secure。HTTP 兼容档位功能可用，但 Cookie 和密码可能被网络窃听，页面必须持续显示不安全传输。
- 页面刷新和 SSE 重连复用同一登录会话。
- 密码错误只返回统一提示，不泄露账户是否存在。
- 连续失败触发渐进式限流。
- 登录页不展示设备和项目缓存。

第一版不要求 OAuth、企业 SSO 和多因素认证，但接口不应把登录方式写死到业务数据模型。

## 4. Web 会话

用户可以：

- 查看当前活跃网页登录会话的设备类型、IP 摘要和最近使用时间。
- 退出当前会话。
- 撤销其他会话。
- 撤销除当前会话外的全部会话。

会话过期时：

- HTTP API 返回未登录。
- SSE 连接关闭。
- 页面保留未发送草稿，但清除敏感缓存并跳转登录页。
- 重新登录后必须重新获取快照，不能继续沿用旧授权判断。

## 5. 设备配对授权

### 5.1 生成配对码

- 只有已登录 Owner 可以生成。
- 配对码默认 10 分钟过期。
- 一次性使用。
- 页面显示过期时间和取消按钮。
- 同一用户最多保留少量未使用配对码，超限时撤销最旧或拒绝创建。

### 5.2 使用配对码

- CLI 提交配对码、设备公钥和基础设备信息。
- 成功后 Server 返回稳定 `device_id` 和短期认证材料。
- 配对码一旦成功消费，任何重复提交均失败。
- 失败不能产生半注册设备。
- HTTP/WS 档位下配对码和初始设备 Token 也会经过未加密网络；CLI 在提交前显示明确警告，推荐只在 VPN/SSH 隧道或可信网络内配对。

### 5.3 撤销

- Owner 撤销设备后，设备凭证立即失效。
- 已存在 WS/WSS 连接被关闭。
- 设备本地文件、项目和 Codex 会话不删除。
- 重新接入必须生成新配对码；不能恢复旧凭证。

## 6. 权限规则

第一版权限矩阵：

| 操作 | 未登录 | Owner Web Session | 有效 Device Credential |
|---|---:|---:|---:|
| 登录 | 是 | 是 | 否 |
| 查看设备/项目摘要 | 否 | 是 | 仅自身同步 |
| 发送远程命令 | 否 | 是 | 否 |
| 建立设备 WS/WSS | 否 | 否 | 是 |
| 上报设备事件 | 否 | 否 | 是 |
| 生成配对码 | 否 | 是 | 否 |
| 撤销设备 | 否 | 是 | 否 |

## 7. 安全与异常体验

- 配对码、会话 Token 和设备 Token 不出现在 URL 和日志中。
- 登录失败、配对失败和凭证过期提供明确但不泄密的错误。
- Server 时间异常时禁止签发过期时间不可信的 Token。
- Cookie 被撤销后，现有 SSE 最迟在短时间内断开。
- 设备时钟偏差不影响长期身份，以 Server 时间为准。

## 8. 第一版不做

- 多用户注册。
- 团队共享设备。
- OAuth/SSO。
- MFA。
- 细粒度项目权限。
- 设备间互相授权。

## 9. 验收标准

1. 未登录用户无法读取任何设备或项目信息。
2. 配对码过期、取消或使用后不能再次使用。
3. 撤销设备后旧 WS/WSS 在限定时间内失效。
4. Web Session 和 Device Credential 不能互换使用。
5. 重复配对请求不会创建多台半重复设备。
6. 登录会话过期后页面重新认证并全量同步。
7. 密码、配对码、Token 和私钥不进入普通日志。
8. HTTP 模式使用正确的非 Secure Cookie 属性并持续告警；HTTPS 模式不会因证书错误自动降级到 HTTP。
