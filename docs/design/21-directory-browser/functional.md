# 远程目录浏览与项目创建：功能设计

> 实现标记：本文描述目标设计；当前 `0.1.0` 已实现项和后续边界以 [后端实现状态](../../implementation-status.md) 为准。

## 1. 目标

用户在手机选择一台在线设备后，可以浏览该设备允许暴露的目录结构，选定一个目录并创建 Project，然后在该 Project 中创建 Thread 和进行多轮对话。

该功能是“目录选择器”，不是远程文件管理器。

## 2. 用户流程

1. 手机进入某台 Device。
2. 点击“添加项目”。
3. 页面显示该设备提供的可浏览根目录。
4. 用户逐层进入子目录。
5. 页面展示目录名、是否为 Git 仓库、是否已有 Project 和是否可选择。
6. 用户选定目录。
7. 输入或确认 Project 显示名称和少量默认配置。
8. 提交创建。
9. Agent 再次验证目录并在本地创建 Project。
10. Server 收到 Project 索引后跳转 Project 页面。

## 3. 允许根目录

Agent 默认提供：

- 当前用户 Home，作为一个可配置根。
- 用户在本地设置中额外添加的 workspace roots。
- 用户显式允许的挂载卷根。

默认不从文件系统 `/` 开始浏览，也不暴露其他系统用户目录。

允许根是“导航边界”，不等于根本身一定可作为 Project。例如 Home 可以用于进入 `~/work`，但 Home 根本身按 Project 危险路径规则显示为不可选择。

本地设置允许：

- 启用/禁用 Home 根。
- 添加或删除额外允许根。
- 控制是否显示隐藏目录。
- 控制是否允许符号链接目标。

## 4. 目录展示

每项只展示：

- 目录名称。
- 安全的路径面包屑。
- 是否有可进入的子目录。
- 是否为 Git 仓库/工作树。
- 是否已登记为 Project。
- 读取权限和可选写权限摘要。
- 是否为符号链接。

第一版不展示文件列表、文件名、文件大小、文件正文和搜索结果。

## 5. 默认排除

即使位于允许根下，也默认排除或不可进入：

- `.ssh`、`.gnupg`、系统钥匙串和凭证目录。
- Nuntius 数据目录。
- Codex 状态目录。
- 浏览器 Profile、云凭证和常见秘密目录。
- Agent 无权访问的目录。
- 系统伪文件系统和设备节点。

排除列表由平台策略提供，用户不能通过手机关闭；需要调整时在设备本地高级设置中进行。

## 6. 在线要求

- 目录浏览要求 Device 处于 online。
- syncing、degraded-storage 或 offline 时不能浏览。
- Server 不长期缓存目录树。
- 页面返回后台后重新进入目录选择器，需要核对 Device 和目录引用是否仍有效。

## 7. 目录引用

页面选中目录时得到 Agent 签发的短期 `directory_ref`：

- 绑定 Device。
- 绑定允许根和规范化路径。
- 有过期时间。
- 不能由 Server 或浏览器修改。
- 只能用于创建 Project，不可用于读取文件。

创建时 Agent 重新验证，不相信之前浏览结果。

## 8. 符号链接和目录变化

- 默认显示符号链接标记。
- 链接目标超出允许根时不可进入或选择。
- 目录在选择后被移动、删除或改成链接时，创建失败并提示重新选择。
- 路径大小写和规范化规则由目标设备平台决定。

## 9. 已有项目

- 已是 active Project 的目录显示“已添加”，点击直接进入原 Project。
- 已 removed Project 可以选择重新激活或创建新记录，具体由 Project 规则处理。
- 同一规范化路径不能创建两个 active Project。

## 10. HTTP 模式提示

HTTP/WS 模式下目录名称和路径面包屑可能被网络窃听。进入选择器前显示不安全传输标记；如果用户不接受，应通过 VPN/SSH 隧道或 HTTPS 使用。

## 11. 第一版不做

- 查看或编辑文件。
- 全盘文件名搜索。
- 上传、下载、移动和删除目录。
- 从手机创建实际目录。
- 切换操作系统用户。
- 远程修改允许根和敏感排除策略。

## 12. 验收标准

1. 手机可以在选定在线 Device 后按层浏览允许目录。
2. Server 不长期保存完整目录树。
3. 页面不能通过修改请求浏览允许根之外的路径。
4. `directory_ref` 过期、篡改或跨设备使用时创建失败。
5. 创建前 Agent 重新 canonicalize 并检查权限。
6. 远程目录 API 不返回文件正文或普通文件列表。
7. 新 Project 正确归属于选定 Device，并可继续创建多个 Thread。
8. 敏感目录和跨根符号链接默认不可访问。
