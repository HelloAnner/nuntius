# 远程移动控制台：功能设计

> 实现标记：本文描述目标设计；当前 `0.1.0` 已实现项和后续边界以 [后端实现状态](../../implementation-status.md) 为准。

## 1. 目标

为手机和平板提供清晰、低摩擦且安全的多设备 Codex 控制体验。重点是快速选择设备和项目、继续对话、查看执行、处理审批，而不是复制桌面 IDE。

## 2. 信息架构

```text
设备
├─ 项目
│  ├─ Thread 列表
│  └─ 对话
├─ 待审批
├─ 设备详情
└─ 诊断摘要

账户
├─ 登录会话
├─ 设备配对
└─ 安全设置
```

移动端底部导航建议：设备、最近、待审批、设置。

每台设备的项目列表包含一个可能出现的“未归类”系统项目；其中 Thread 历史可读，但没有 cwd，必须回设备本地归类后才能继续执行。

## 3. 设备首页

- 在线设备置前。
- 显示在线/同步/降级/离线、最后在线和版本警告。
- 卡片展示项目数、活动 Turn、待审批。
- 点击设备进入项目列表。
- 离线设备仍可查看 Server 已同步的完整历史，实时状态明确 stale。

## 4. 项目页

- 展示项目名称、分支、dirty 和可用状态。
- 点击“添加项目”后浏览在线设备允许的目录结构，选择目录并创建 Project。
- 查看最近 Thread。
- 在 active Project 新建 Thread。
- 修改允许的显示配置。
- 不提供任意路径输入和文件浏览。

## 5. Thread 列表

- “最近/全部会话”入口直接从 Server 聚合所有设备，支持再按 Device 和 Project 筛选。
- 按最近活动排序。
- 区分执行中、等待审批、空闲和归档。
- 支持标题搜索和 cursor 分页。
- 离线时仍显示 Server 完整历史和最后同步位置，但不能继续执行。
- 快速继续最近 Thread。
- 每个 Thread 显示历史完整度、最后同步时间、所属 Device 和 Project；设备离线只禁用执行，不隐藏历史。

## 6. 对话页

- 顶部固定展示设备、项目和连接状态。
- 消息流显示用户、Agent、命令、工具、文件和审批 Item。
- 输入区支持发送文本。
- Turn 运行时支持 steer 和 interrupt。
- 页面滚动时新事件不强制抢夺位置，提供“回到底部”。
- 长命令输出折叠、按需加载。
- unknown 状态突出显示“核对状态”，不提供无脑重发。

## 7. 审批体验

- 全局待审批入口显示数量。
- 审批卡始终显示目标设备/项目。
- 高风险操作二次确认。
- 页面从后台恢复后先刷新审批状态。
- 多标签页处理后同步终态。

## 8. 连接和恢复

- 顶部用轻量状态显示 live/reconnecting/syncing/stale。
- 短暂重连不清空页面。
- 从后台恢复时自动同步。
- Server 可达但设备离线时，明确区分两者。
- 用户草稿保存在当前设备/Thread 作用域，不因重连丢失。
- HTTP/WS 档位在全站持续显示不可关闭的不安全传输横幅；它不阻止核心功能，但不能让用户误以为登录等于加密。

## 9. 移动端设计约束

- 触控目标足够大。
- 不依赖 hover。
- 键盘弹起时输入框可见。
- 支持横屏平板双栏：Thread 列表 + 对话。
- 支持深色模式。
- 重要状态不只依靠颜色表达。
- 首屏不加载完整 Thread 历史。
- 历史按 Turn/Item cursor 分页加载，用户可以继续向上加载直至完整记录。

## 10. PWA 边界

- HTTPS 或浏览器认可的 localhost 环境可以安装到桌面并缓存静态资源。
- 普通远程 HTTP 下不注册 Service Worker、不宣称可安装 PWA，但对话、历史、目录选择和控制功能保持可用。
- 离线时只展示最近缓存和明确离线状态。
- 不在离线状态保存待自动执行命令。
- 第一版不依赖后台 Service Worker 保持实时连接。
- Web Push 作为后续审批通知能力。

## 11. 验收标准

1. 用户在三步内从设备进入项目 Thread 并发送消息。
2. 页面始终明确当前目标设备和项目。
3. 手机锁屏/切网后恢复页面能重新同步。
4. 离线缓存不会被误认为实时状态。
5. 长对话和大量 delta 不造成明显内存持续增长。
6. 审批在小屏幕上仍显示足够上下文。
7. 页面具备基础键盘导航、读屏标签和非颜色状态表达。
8. HTTP 与 HTTPS 核心功能一致，但 HTTP 永久显示风险且不启用 secure-context 专属能力。
9. 未归类 Thread 远程可读、无 cwd、不可执行。
10. 用户无需先进入某台在线设备，就能从“全部会话”读取所有设备已同步历史。
