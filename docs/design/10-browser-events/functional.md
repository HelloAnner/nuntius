# 浏览器 SSE 事件流：功能设计

> 实现标记：本文描述目标设计；当前 `0.1.0` 已实现项和后续边界以 [后端实现状态](../../implementation-status.md) 为准。

## 1. 目标

为手机、平板和本地页面提供可自动重连、可识别缺口、可通过快照恢复的实时事件流。事件流负责“及时”，快照负责“正确”。

SSE 可运行在 HTTPS 或 HTTP 上；两种模式使用相同事件 ID、重放和恢复协议。HTTP 模式本身不加密，页面必须继承全局不安全传输提示。

## 2. 用户体验

页面连接状态：

```text
connecting -> live
live -> reconnecting -> syncing -> live
任意 -> stale/offline
```

- 首次进入显示轻量同步状态，不阻塞页面骨架。
- 短暂网络抖动时保留已有内容，显示重连提示。
- 恢复后先补事件或请求快照，再恢复 live。
- 不能因为 SSE 断开就把正在运行的 Turn 标记失败。

## 3. 事件范围

一个浏览器标签页维持一条用户级 SSE：

- 设备状态。
- 项目和 Thread 摘要变化。
- 当前或订阅 Thread 的 Turn/Item 事件。
- 审批。
- 命令状态。
- Server 通知和 resync 指令。
- 历史同步进度、完整度变化和新持久化 Item 通知；正文仍按 HTTP 分页快照读取。

高流量 Thread 只有在页面订阅或用户明确关注时推送详细 delta；其余 Thread 只推送生命周期和摘要。

## 4. 订阅行为

- 登录后建立基础用户事件流。
- 进入 Thread 时通过 HTTP 更新该标签页的详细订阅范围。
- 离开 Thread 后取消详细 delta，但保留完成、失败和审批事件。
- 多标签页独立订阅，不相互抢占。

## 5. 重连

- 每条事件有 `id`。
- 浏览器使用原生 EventSource 自动重连。
- Server 根据 `Last-Event-ID` 尝试补发。
- 超出重放范围时发送 `resync_required`。
- 页面收到后关闭旧流、请求 sync 快照、重新订阅。

## 6. 慢消费者

当页面处理太慢：

- 先合并普通文本 delta。
- 保留生命周期、审批和错误事件。
- 队列超过硬限制时发送 resync_required 并断开。
- 页面重新获取聚合快照，不要求补回每一个视觉 delta。

## 7. 页面后台

- 页面进入后台后不假设连接持续有效。
- 恢复可见时检查最后同步时间。
- 超过阈值先请求轻量 sync。
- 未完成同步前，高风险审批按钮禁用。

## 8. 错误体验

- 未登录：跳转登录，不无限重连。
- Server 维护：显示重连中，遵守 retry 提示。
- 游标过期：显示重新同步，不显示数据丢失恐慌提示。
- 协议不兼容：要求刷新/升级前端资源。

## 9. 验收标准

1. 网络切换后页面能从事件 ID 继续或自动重同步。
2. 重复事件不会重复追加 Agent 文本。
3. SSE 断开不会改变 Turn 业务状态。
4. 慢页面不会导致 Server 内存无限增长。
5. 审批和终态事件优先于普通 delta。
6. 页面后台恢复后先同步再开放危险操作。
7. 多标签页同时使用时互不抢占事件。
8. HTTP 和 HTTPS 下 SSE 恢复语义一致，HTTP 页面持续显示传输风险。
9. 设备离线时 SSE 断开不影响 Server 历史分页读取。
