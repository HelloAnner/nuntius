# 本地控制台：功能设计

> 实现标记：本文描述目标设计；当前 `0.1.0` 已实现项和后续边界以 [后端实现状态](../../implementation-status.md) 为准。

## 1. 目标

让用户在当前工作电脑的浏览器中完成设备初始化、项目管理、Codex 会话操作、审批和诊断。公网 Server 不可用或设备尚未配对时，本地控制台仍是完整的本机操作入口。

## 2. 打开方式

- 运行 `nuntius open`。
- 若 Agent 未运行，提示启动或按平台策略启动。
- CLI 输出并打开一次性本地 URL。
- 页面只连接 loopback，不通过公网绕回本机。

## 3. 信息架构

```text
本机概览
├─ 项目
│  └─ Thread
│     └─ 对话/执行/审批
├─ 未归类 Thread
├─ 连接与配对
├─ Codex 状态
├─ 设置
└─ 诊断
```

## 4. 本机概览

展示五层状态：

- Agent 是否运行。
- 本地数据库是否健康。
- Codex App Server 是否 ready。
- 公网 Server 是否已连接。
- 当前项目、活动 Turn 和待审批摘要。

每层提供简短状态、最近错误和对应处理动作。

## 5. 项目管理

- 添加本地目录。
- 修改显示名称和默认配置。
- 校验路径。
- 查看 Git 摘要。
- 暂停、恢复和移除索引。
- 打开 Project 的 Thread 列表。

只有本地控制台可以添加任意新路径。移除按钮明确说明“不删除文件”。

## 6. Thread 与对话

- 新建、恢复、归档和取消归档。
- 查看完整本机可获得的历史。
- 发送文字、Steer 和 Interrupt。
- 查看 Agent 消息、命令、工具和文件事件。
- 处理审批。
- 查看 `unknown` 操作并执行状态核对。

本地页面与远程页面使用相同消息组件和状态词汇。

## 7. 未归类 Thread

- 展示 App Server 已存在但无法关联 Project 的 Thread。
- 显示安全的 cwd 提示。
- 允许用户选择一个已有 Project 进行关联。
- 不自动创建新的 Project。
- 未归类 Thread 会以无 cwd 的系统 Project 归属同步完整规范化历史；只有本地页面能把它重新关联到真实 Project。

## 8. 配对与连接

- 查看当前 Server URL 和配对状态。
- 输入或粘贴一次性配对码。
- 查看 WS/WSS 连接阶段、传输安全状态、上次连接和重试时间。
- 手动重连。
- 解除本地配对。

解除本地配对不删除项目和会话数据。

## 9. Codex 状态

- Codex 是否安装。
- App Server 版本和兼容性。
- 登录状态摘要。
- 启动、停止、重启 App Server。
- 需要登录时展示本机操作指引。

不展示或导出 Codex Token。

## 10. 设置

第一版设置：

- Server URL。
- 开机启动。
- 日志级别和保留期。
- 本地事件保留和磁盘上限。
- 手机目录浏览允许根、隐藏目录和符号链接策略。
- App Server 二进制路径。
- 是否向 Server 上传绝对路径提示，默认关闭。

危险设置要求重新验证并展示影响。

## 11. 诊断

- 一键刷新健康状态。
- 查看最近脱敏错误。
- 运行 doctor。
- 下载/保存脱敏诊断包。
- 查看 inbox/outbox 是否积压。
- 查看版本兼容建议。

## 12. 验收标准

1. 未配对和公网离线时可管理本机项目与 Codex。
2. 用户可在本地直接添加目录；远程页面只能通过本机允许根和短期目录引用添加。
3. 所有本地审批和远程审批使用同一 approval 状态。
4. App Server 未 ready 时诊断页面仍可访问。
5. 页面明确区分本地状态和公网连接状态。
6. 解除配对不删除本地项目、Thread 或文件。
7. 未归类 Thread 可在本地安全整理。
8. 未归类历史在远程可读，但不暴露 cwd 且不能继续执行。
