# 安全：功能设计

> 实现标记：本文描述目标设计；当前 `0.1.0` 已实现项和后续边界以 [后端实现状态](../../implementation-status.md) 为准。

## 1. 目标

保护远程控制能力不被未授权用户、恶意网页、被盗设备凭证或重放消息滥用。Nuntius 能触发本地代码读取、命令执行和文件修改，因此安全失败必须默认拒绝，而不是为了可用性放行。

## 2. 信任模型

- Owner 信任自己部署的公网 Server 负责身份、路由以及规范化完整会话历史存储；Server 管理员和数据库/备份操作者能够接触这些历史。
- Owner 信任已配对 Device Agent 在本机权限范围内工作。
- 手机浏览器不直接信任设备，只通过 Server 授权。
- Server 不获得 Codex 本地凭证。
- Agent 不接受来自 Server 的任意 shell，只接受定义好的 Nuntius Command。
- App Server 权限仍受 Codex sandbox 和 approval policy 控制。

## 3. 用户安全能力

- 查看和撤销网页登录会话。
- 查看、命名和撤销设备。
- 查看最近安全相关事件。
- 高风险审批获得足够上下文和二次确认。
- 查看 Server/Agent 是否使用安全连接。
- 在设备丢失时一键撤销该设备。
- 在怀疑泄露时撤销全部网页登录会话。

## 4. 安全默认值

- 公网生产默认使用 HTTPS/WSS，并拒绝静默降级。
- 为无 TLS 环境提供显式 HTTP/WS 兼容档位；非 loopback 必须设置 `allow_insecure_http=true`，CLI 和页面持续显示风险。该档位保证功能兼容，不保证防窃听、防篡改或 Server 身份真实性。
- 本地服务只绑定 loopback。
- 远程只能使用 Agent 受控目录浏览器和短期目录引用创建项目，不能提交任意路径字符串。
- 设备离线不排队长期副作用命令。
- 不提供远程“始终批准所有操作”。
- 不上传 Codex Token 和项目文件；完整规范化会话历史会上传，因此 Server 和备份必须按敏感数据保护。
- 日志默认脱敏。
- 协议未知危险命令默认拒绝。

## 5. 威胁场景

### 账户被盗

攻击者可能控制所有在线设备。缓解：强密码、限流、会话列表、撤销、后续 MFA。

### 配对码泄露

攻击者可能注册伪设备。缓解：短期、一次性、限流、展示待接入设备信息、可立即撤销。

### 设备凭证泄露

攻击者可能冒充 Agent。缓解：设备密钥对、系统钥匙串、短期 Token、key version 和撤销。

### 恶意网页访问 localhost

可能通过 DNS rebinding/CSRF 控制本地 Agent。缓解：Host/Origin、随机本地 Session、CSRF、loopback only。

### 重放旧命令

可能重复执行副作用。缓解：command ID、idempotency、expires_at、inbox 去重、连接 epoch。

### Prompt/输出注入 UI

Agent 输出可能包含 HTML 或诱导内容。缓解：纯文本、安全 Markdown、审批区与模型文本视觉隔离。

### HTTP/WS 被监听或篡改

攻击者可能窃取网页登录 Cookie、设备 Bearer Token、完整会话历史、目录名称或修改控制命令。缓解：默认 HTTPS/WSS；无证书时优先使用 VPN/SSH 隧道；显式风险开关和永久告警。第一版不声称应用层登录或签名能够替代 TLS，也不在 HTTP 模式启用依赖 secure context 的能力。

### 目录越权与符号链接竞态

攻击者可能伪造路径、重放旧选择或利用 symlink 把 Project 指向允许根之外。缓解：只返回受控根、签名短期 `directory_ref`，创建时重新 canonicalize 并验证归属与根边界，拒绝客户端提交任意远程路径。

### 历史聚合泄露

Server 数据库、备份、导出或日志可能泄露对话。缓解：最小权限、静态加密/加密备份、正文不入日志、字段白名单、删除与保留策略；项目源文件和 Codex 凭证不进入历史存储。

## 6. 安全事件体验

- 认证失败不展示内部详情。
- Device revoked、重复连接和协议攻击进入安全审计。
- 可疑情况下设备状态变为 degraded/revoked，不自动恢复授权。
- 安全错误给出恢复步骤，如重新登录、重新配对、检查 Server 证书。

## 7. 第一版边界

- Server 是可信中继，第一版不实现端到端加密。
- 不做企业合规、组织策略和审计导出平台。
- 不做远程设备 wipe。
- 不承诺防御已完全控制用户电脑的本地攻击者。

## 8. 验收标准

1. 未授权网页无法通过 localhost 控制 Agent。
2. 设备撤销后不能建立或维持 WS/WSS。
3. 旧/过期命令不会执行。
4. App Server/Codex 凭证不上传 Server。
5. 模型输出不能注入可执行前端 HTML。
6. 敏感字段不进入普通日志和诊断包。
7. 安全校验依赖失败时默认拒绝。
8. HTTP/WS 模式与安全模式功能一致但始终显示不安全传输，且不会设置无效的 Secure Cookie。
9. 伪造、过期或跨设备的 `directory_ref` 无法创建 Project。
10. 完整消息可进入 History Store，但凭证、任意源文件正文和秘密字段不会进入日志或历史 DTO。
